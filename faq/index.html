Q: How do I add Debian packages to my project?

To add a Debian package to your Emily project, follow these steps:

  1. Locate the directory of your project. If you are unsure where you put your Emily project, you can use the command emily list -f to view all your projects. The location of each project will be shown in the “PATH” column.
  2. Open the project’s Dockerfile.
    • If you are using VS Code, this is located in the .devcontainer folder.
    • If you are using PyCharm, this is located in the .idea folder.
    • If you are using Jupyter, this is located in the .jupyter folder.
  3. Insert the command RUN apt-get -y update && apt-get install -y <package name> in the Dockerfile just below the line that says “# add installation of Debian packages here”.
    • For example, add RUN apt-get -y update && apt-get install -y git if you want to add git to your project.
    • You can also add more than one package at a time. If you want both GCC and CMake, add RUN apt-get -y update && apt-get install -y gcc cmake to the Dockerfile.
    • Some packages are not directly available with apt-get and must be downloaded first. For example, to install nodejs in your Emily project, first use curl to download it and then apt-get to install, by adding the following to your Dockerfile:
    RUN curl -sL https://deb.nodesource.com/setup_12.x | bash - 
RUN apt-get -y update && apt-get install -y nodejs
  4. Open your project with the emily open <project name> command.

For more information about modifying the Dockerfile, check out the Docker documentation.

Q: How do I make Emily detect my PyCharm professional installation?

If you experience this issue on Windows, it is most likely because Emily cannot find the PyCharm installation in your PATH environment variable. You can use this guide to add the folder containing your PyCharm installation to the PATH variable.

If you have installed PyCharm using JetBrains toolbox, you will have to setup the command-line interface version by following this guide.

Note: Emily expects PyCharm professional to be on a path that includes a folder named “JetBrains”. If the path to your PyCharm folder does not contain a “JetBrains” folder, Emily will not be able to find it.

Q: I get an error when running brew cleanup. How do I fix this?

This is a known issue with brew. The most likely fix is to run

sudo chown -R $(whoami):admin /usr/local/* \
&& sudo chmod -R g+rwx /usr/local/*

to reset the permissions in /usr/local. You can then reinstall packages by running brew list | xargs brew reinstall.

If this does not fix your issue, see this thread for other possible solutions.
